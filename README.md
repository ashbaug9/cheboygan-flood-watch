# Cheboygan Flood Watch — GitHub Pages Deploy

## Files
- `index.html` — Landing page (homepage)
- `cheboygan_flood_map_v3.html` — Desktop interactive map
- `mobile.html` — Mobile-optimized map
- `flood_zone_map.html` — Printable aerial zone map (partial breach)

---

## Deploy to GitHub Pages (5 minutes)

### Step 1 — Create a GitHub account
Go to **github.com** and sign up if you don't have one. Free.

### Step 2 — Create a new repository
1. Click the **+** icon (top right) → **New repository**
2. Name it: `cheboygan-flood-watch` (or anything you want)
3. Set visibility to **Public**
4. Leave everything else unchecked
5. Click **Create repository**

### Step 3 — Upload the files
1. On the new repo page, click **uploading an existing file** (the link in the middle of the page)
2. Drag all four `.html` files into the upload area (not the zip — extract it first)
3. Scroll down, click **Commit changes**

### Step 4 — Enable GitHub Pages
1. Click **Settings** tab (top of repo page)
2. In the left sidebar, click **Pages**
3. Under **Source**, select **Deploy from a branch**
4. Under **Branch**, select `main` and `/ (root)`, click **Save**

### Step 5 — Get your URL
- GitHub will show a banner: *"Your site is live at..."*
- URL format: `https://YOUR-USERNAME.github.io/cheboygan-flood-watch/`
- Takes about 60 seconds to go live after saving

---

## Updating files later

1. Go to your repo on github.com
2. Click the file you want to update
3. Click the **pencil icon** (Edit)
4. Paste new content, click **Commit changes**
5. Site updates in ~30 seconds

Or drag new files to the repo's main page (same upload flow as Step 3).
